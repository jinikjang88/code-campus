# GitHub 업로드 가이드

## 새 저장소 생성 및 업로드

### 1. GitHub에서 새 저장소 생성
1. GitHub.com 접속
2. 우측 상단 `+` → `New repository` 클릭
3. Repository name: `code-campus` (또는 원하는 이름)
4. Description: "신입 개발자를 위한 교육용 2.5D 게임"
5. Public 선택
6. **"Initialize this repository with a README"는 체크하지 않기**
7. `Create repository` 클릭

### 2. 로컬에서 Git 초기화 및 푸시

```bash
# 프로젝트 폴더로 이동
cd code-campus

# Git 초기화
git init

# 모든 파일 추가
git add .

# 첫 커밋
git commit -m "Chapter 1: 캠퍼스 탐험 - 기본 게임 시스템 구현"

# 기본 브랜치 이름을 main으로 변경
git branch -M main

# GitHub 저장소 연결 (your-username을 실제 사용자명으로 변경)
git remote add origin https://github.com/your-username/code-campus.git

# 푸시
git push -u origin main
```

## 추가 기능 (선택사항)

### GitHub Pages로 배포하기

```bash
# gh-pages 브랜치 생성 및 배포
npm run build
git add dist -f
git commit -m "Deploy to GitHub Pages"
git subtree push --prefix dist origin gh-pages
```

그 후 GitHub 저장소 Settings → Pages에서:
- Source: Deploy from a branch
- Branch: gh-pages / root
- Save

배포 URL: `https://your-username.github.io/code-campus/`

### 태그 추가하기

```bash
# Chapter 1 태그 생성
git tag -a v1.0.0 -m "Chapter 1: Campus Exploration"
git push origin v1.0.0
```

```
├── CHANGELOG.md

```

## 커밋 메시지 컨벤션 (권장)

```
feat: 새로운 기능 추가
fix: 버그 수정
docs: 문서 변경
style: 코드 포맷팅 (기능 변경 없음)
refactor: 코드 리팩토링
test: 테스트 코드 추가
chore: 빌드 프로세스 또는 도구 변경
```

예시:
```bash
git commit -m "feat: NPC 대화 시스템 추가"
git commit -m "fix: 플레이어 이동 버그 수정"
git commit -m "docs: README에 설치 가이드 추가"
```
